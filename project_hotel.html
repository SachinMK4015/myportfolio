Hotel Data Analytics Project

1. Project Overview

This project involved analyzing hotel booking data to uncover trends, optimize pricing strategies, and improve customer satisfaction. The aim was to provide actionable insights that could help hotel management make data-driven decisions.

2. Dataset

The project involved the use of five distinct CSV files, each serving a specific purpose in the analysis:

  • dim_date.csv: This file contains the date dimension data, providing detailed information about the dates, including the year, month, day, and other relevant date-related attributes.
  • dim_hotels.csv: This file contains information about the hotels, including hotel IDs, names, locations, and other descriptive attributes that uniquely identify and characterize each hotel in the dataset.
  • dim_rooms.csv: This file contains data related to the room types available in each hotel, including room ID, room type, capacity, and other attributes that describe the various room options offered.
  • fact_aggregated_bookings.csv: This fact table contains aggregated booking data, including metrics such as total bookings, revenue generated, and average occupancy rates, aggregated by various dimensions like date, hotel, and room type.
  • fact_bookings.csv: This file contains detailed booking data, including individual booking records with information about booking IDs, customer IDs, booking dates, room IDs, and financial details such as revenue generated and revenue realized.

3. Methodology

Exploratory Data Analysis (EDA)

We conducted an exploratory data analysis to identify key trends in the data. Tools such as Pandas, Matplotlib, and Seaborn were used to create visualizations like histograms, heatmaps, and time series plots.

For example, a bar graph is generated to show the bookings done by different platforms:

import seaborn as sns
import matplotlib.pyplot as plt
platform_counts = df_bookings.booking_platform.value_counts()
sns.set_style("whitegrid")
colors = sns.color_palette("pastel", len(platform_counts))
plt.figure(figsize=(10, 6))
platform_counts.plot(kind="bar", color=colors, edgecolor='black')
plt.title('Frequency of Bookings by Platform', fontsize=16, fontweight='bold', pad=20)
plt.xlabel('Booking Platform', fontsize=14, fontweight='bold')
plt.ylabel('Number of Bookings', fontsize=14, fontweight='bold')
plt.xticks(rotation=45, ha='right', fontsize=12)
plt.grid(axis='y', linestyle='--', alpha=0.7)
plt.tight_layout()
plt.show()

4. Insights & Analysis

Key Findings

  • Platform Booking Frequency: The analysis revealed that "Make Your Trip" and other similar platforms recorded the highest frequency of bookings, indicating their strong market presence and customer preference.
  • Revenue Realized: The "Presidential Suites" category generated the highest revenue, with a maximum value recorded at ₹45,220, highlighting its significance in the hotel’s revenue stream.
  • Average Occupancy by Room Class: The average occupancy rates for different room classes were as follows:
    • Elite: 58.01%
    • Premium: 58.03%
    • Presidential: 59.28%
    • Standard: 57.89%
    The "Presidential" room class had the highest average occupancy, suggesting strong demand for luxury accommodations.
  • Average Occupancy Rate by City: The average occupancy rates across key cities were:
    • Bangalore: 56.33%
    • Delhi: 61.51%
    • Hyderabad: 58.12%
    • Mumbai: 57.91%
    Delhi recorded the highest average occupancy rate, indicating a robust market for hotel stays in the capital.
  • Occupancy Rates by Day Type: The occupancy rates were significantly higher during weekends compared to weekdays:
    • Weekday: 51.81%
    • Weekend: 73.96%
    This trend suggests that weekends are the peak period for hotel stays, likely due to leisure travel and events.

Business Implications

The insights from this analysis suggest that hotel management could improve revenue by dynamically adjusting room rates based on demand patterns. Additionally, targeted marketing campaigns could be developed for different customer segments.

5. Challenges & Solutions

Challenges

One of the primary challenges encountered during the project was managing inaccuracies in the revenue-related data, specifically in the "Revenue Generated" and "Revenue Realized" fields. These discrepancies could have significantly skewed the analysis, leading to misleading conclusions. Addressing this issue was critical to ensure the integrity and reliability of the insights derived from the data.

Solutions

We used imputation techniques to fill in missing values and removed records that were incomplete beyond a certain threshold. Additionally, we employed statistical tests to ensure the validity of the imputed data.
